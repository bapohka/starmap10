# Инструкция по запуску StarMap10

## Часть 1: Запуск через Docker

### Что такое Docker?

Docker — это программа, которая упаковывает твоё приложение вместе со всем окружением (Node.js, зависимости и т.д.) в изолированный «контейнер». Это как виртуальная машина, но легче и быстрее. Контейнер работает одинаково на любом компьютере.

### Шаг 1. Установи Docker Desktop

1. Перейди на https://www.docker.com/products/docker-desktop/
2. Скачай версию для Windows
3. Запусти установщик, следуй инструкциям (всё по умолчанию)
4. После установки перезагрузи компьютер
5. Запусти Docker Desktop — в трее (справа внизу) появится иконка кита
6. Подожди, пока Docker Desktop покажет статус «Running» (зелёный индикатор)

### Шаг 2. Создай файл Dockerfile

В корне проекта (рядом с `package.json`) создай файл с именем `Dockerfile` (без расширения):

```dockerfile
# Этап 1: сборка проекта
FROM node:20-alpine AS build
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci
COPY . .
RUN npm run build

# Этап 2: раздача через nginx
FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

**Что тут происходит:**
- Первый этап (`build`) — берёт образ с Node.js, устанавливает зависимости, собирает проект командой `npm run build`. Результат — папка `dist` с готовыми HTML/JS/CSS файлами.
- Второй этап — берёт лёгкий веб-сервер nginx и копирует в него собранные файлы. Nginx будет отдавать их браузеру.

### Шаг 3. Создай файл .dockerignore

В корне проекта создай файл `.dockerignore` (чтобы не копировать лишнее в контейнер):

```
node_modules
dist
.git
```

### Шаг 4. Собери Docker-образ

Открой терминал в папке проекта и выполни:

```bash
docker build -t starmap10 .
```

- `docker build` — команда сборки образа
- `-t starmap10` — даём образу имя «starmap10»
- `.` — говорим Docker искать Dockerfile в текущей папке

Первая сборка займёт пару минут (скачиваются базовые образы). Следующие сборки будут быстрее.

### Шаг 5. Запусти контейнер

```bash
docker run -d -p 8080:80 --name starmap10 starmap10
```

- `-d` — запуск в фоне (detached)
- `-p 8080:80` — пробрасываем порт: localhost:8080 на твоём компьютере → порт 80 внутри контейнера
- `--name starmap10` — имя контейнера (для удобства)

### Шаг 6. Открой в браузере

Перейди по адресу: **http://localhost:8080**

Готово! Приложение работает в Docker.

### Полезные команды Docker

| Команда | Что делает |
|---------|-----------|
| `docker ps` | Показать запущенные контейнеры |
| `docker stop starmap10` | Остановить контейнер |
| `docker start starmap10` | Запустить остановленный контейнер |
| `docker rm starmap10` | Удалить контейнер |
| `docker logs starmap10` | Посмотреть логи контейнера |
| `docker build -t starmap10 .` | Пересобрать образ после изменений в коде |

**Если меняешь код**, нужно пересобрать образ и пересоздать контейнер:

```bash
docker stop starmap10
docker rm starmap10
docker build -t starmap10 .
docker run -d -p 8080:80 --name starmap10 starmap10
```

## Часть 2: Деплой на Vercel

### Что такое Vercel?

Vercel — это облачная платформа для хостинга сайтов. Ты загружаешь код, Vercel сам его собирает и публикует по публичному URL. Бесплатный тариф покрывает личные проекты.

### Предварительное требование: GitHub

Vercel работает через GitHub (или GitLab/Bitbucket). Если у тебя ещё нет репозитория на GitHub:

1. Зарегистрируйся на https://github.com (если ещё нет аккаунта)
2. Установи Git: https://git-scm.com/downloads
3. Создай репозиторий на GitHub (кнопка «New repository»)
4. Свяжи локальный проект с GitHub:

```bash
git init
git add .
git commit -m "feat: initial commit"
git remote add origin https://github.com/ТВОЙ_ЛОГИН/starmap10.git
git branch -M main
git push -u origin main
```

### Шаг 1. Зарегистрируйся на Vercel

1. Перейди на https://vercel.com
2. Нажми «Sign Up»
3. Выбери «Continue with GitHub» — это самый простой способ
4. Разреши Vercel доступ к твоему GitHub

### Шаг 2. Импортируй проект

1. На главной странице Vercel нажми **«Add New...» → «Project»**
2. Найди репозиторий `starmap10` в списке и нажми **«Import»**

### Шаг 3. Настрой сборку

Vercel автоматически определит, что это Vite-проект. Проверь настройки:

| Поле | Значение |
|------|---------|
| Framework Preset | Vite |
| Build Command | `npm run build` |
| Output Directory | `dist` |
| Install Command | `npm install` |

Обычно всё определяется автоматически и менять ничего не нужно.

### Шаг 4. Нажми Deploy

Нажми кнопку **«Deploy»** и подожди 1–2 минуты.

После завершения ты получишь публичный URL вида:
**https://starmap10-xxxxx.vercel.app**

Этот URL доступен из любой точки мира.

### Автоматические деплои

После первоначальной настройки Vercel **автоматически** пересобирает и публикует сайт при каждом `git push` в ветку `main`. То есть рабочий процесс такой:

1. Вносишь изменения в код
2. Коммитишь и пушишь:
   ```bash
   git add .
   git commit -m "feat: добавил новую фичу"
   git push
   ```
3. Vercel сам подхватит изменения и задеплоит — через пару минут обновлённый сайт будет доступен

### Полезные возможности Vercel

- **Preview Deployments** — если создать Pull Request на GitHub, Vercel автоматически создаст отдельный превью-URL для этого PR. Удобно для проверки изменений перед мержем.
- **Логи сборки** — на дашборде Vercel можно посмотреть логи, если сборка упала.
- **Кастомный домен** — в настройках проекта (Settings → Domains) можно привязать свой домен вместо `.vercel.app`.
- **Переменные окружения** — если понадобятся env-переменные, их можно задать в Settings → Environment Variables.

## Сравнение: Docker vs Vercel

| | Docker | Vercel |
|---|--------|--------|
| **Для чего** | Локальный запуск, свой сервер | Публикация в интернет |
| **Нужен свой сервер?** | Да (или локально) | Нет, всё в облаке |
| **Стоимость** | Бесплатно локально | Бесплатно (Hobby план) |
| **Автодеплой** | Нет (вручную) | Да, при git push |
| **Когда использовать** | Тестирование, CI/CD, свой VPS | Быстрый деплой статического сайта |
